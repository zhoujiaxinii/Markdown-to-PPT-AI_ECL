# 匹配规则说明

> 更新日期：2026-03-06
> 基于 3.6 模板.pptx (32页)
> 详细说明MD内容如何匹配到PPT模板的规则

---

## 一、PPT结构

生成的PPT结构如下：

```
1. 封面页 - 课程标题 ({{h0_0}}) + 1图片
2. 目录页 - 列出所有章节 + 2图片
3. 章节页 - 每个二级标题一页 ({{h1_0}})
4. 正文页 - 每个三级标题一页 ({{h2_0}} + {{h3_*}})
5. ...
6. 结束页 - 谢谢（自动添加，无需MD编写） + 2图片
```

---

## 二、智能模板匹配

### 2.1 匹配原则

**核心要求**：根据MD内容精确匹配模板，清除多余占位符

| MD内容 | 匹配模板 | 说明 |
|--------|---------|------|
| 课程标题 | 封面模板 | `{{h0_0}}` |
| 章节列表 | 目录模板 | 自动填充 |
| 二级标题 | 章节模板 | `{{h1_0}}` |
| 三级标题 | 正文模板 | 根据文本框数+图片数匹配 |
| 含音频 | 音频模板 | 自动识别 `{{audio}}` |
| 含视频 | 视频模板 | 自动识别 `{{video}}` |
| N个文本框 | 文本框数≥N的最小模板 | 清除未使用的占位符 |
| M个图片 | 图片数=M的模板 | 精确匹配图片数 |

### 2.2 页面生成顺序

```
1. 解析MD文档
   ├─ 提取课程标题 (h0)
   ├─ 提取章节列表 (h1)
   └─ 提取小节内容 (h2 + content + audio/video)

2. 生成PPT页面
   ├─ 封面页 (h0)
   ├─ 目录页 (h1列表)
   ├─ 遍历章节
   │   ├─ 章节页 (h1标题)
   │   └─ 正文页 (h2内容)
   └─ 结束页 (自动添加)

3. 清理页面
   └─ 清除未使用的占位符
```

---

## 三、占位符规则

### 3.1 文本占位符

| 占位符 | 用途 |
|--------|------|
| `{{h0_0}}` | 封面标题 |
| `{{h1_0}}` | 章节标题 |
| `{{h2_0}}` | 小节标题 |
| `{{h3_0}}`~`{{h3_3}}` | 文本内容（最多4个） |

### 3.2 媒体占位符

| 占位符 | 用途 |
|--------|------|
| `{{audio}}` | 音频图标位置 |
| `{{video}}` | 视频播放器位置 |

### 3.3 模板匹配示例

| 小节内容 | 匹配模板 |
|---------|---------|
| 1个文本框 | 单文本框模板 |
| 2个文本框 | 双文本框模板 |
| 3个文本框 | 三文本框模板 |
| 4个文本框 | 四文本框模板 |
| 含音频 | 匹配带 `{{audio}}` 的模板 |
| 含视频 | 匹配带 `{{video}}` 的模板 |

## 四、章节页处理

### 4.1 章节识别

MD中的二级标题 (`##`) 会被识别为章节：

```markdown
## 白色    ← 章节页：白色
### 学一学  ← 正文页
### 词汇    ← 正文页

## 红色    ← 章节页：红色
### 学一学  ← 正文页
```

### 4.2 章节页生成

- 每个二级标题生成一个章节页
- 使用模板中的 `{{h1_0}}` 占位符
- 章节名称自动添加拼音

---

## 五、目录页处理

### 5.1 目录生成

- 自动从所有章节 (`h1`) 提取标题
- 填充到目录模板的对应位置
- 支持多个章节的目录项

### 5.2 目录模板

目录页应该包含可以填充章节名的占位符，如：
- `{{h1_0}}` - 第1个章节
- `{{h1_1}}` - 第2个章节
- `{{h1_2}}` - 第3个章节
- ...

---

## 六、结束页处理

### 6.1 自动添加

- PPT末尾自动添加结束页
- 无需在MD中编写
- 模板中应包含"谢谢"或"xiè xie"字样

### 6.2 模板识别

结束页通过以下方式识别：
- 页面中包含"谢谢"文字
- 页面中包含"xiè xie"拼音

---

## 七、拼音生成规则

### 7.1 垂直对齐原则
- **核心要求**：每个拼音精确对齐到对应汉字的**正上方**
- 拼音和汉字**一一对应**

### 7.2 表格实现方案
使用 **2行 x N列** 的无边框、无底色表格：

```
原文：这是白色
实现：
 zhè  shì  bái  sè
 这    是    白    色
```

### 7.3 字符间距规则
- **拼音间距**：每个拼音后 + 1个字母宽度
- **汉字间距**：跟随对应拼音
- **列宽计算**：动态计算，不均分

### 7.4 自动缩放
- 文本过长时自动缩小字号
- 最小字号：10号

---

## 八、音视频处理

### 8.1 音频识别

MD中以 `.mp3`、`.wav`、`.m4a` 结尾的段落被识别为音频：
```markdown
这是一个音频：audio/lesson1.mp3
```

### 8.2 视频识别

MD中以 `.mp4`、`.avi`、`.webm` 结尾的段落被识别为视频：
```markdown
这是一个视频：video/lesson1.mp4
```

### 8.3 模板匹配

- 包含音频的正文页会匹配带 `{{audio}}` 占位符的模板
- 包含视频的正文页会匹配带 `{{video}}` 占位符的模板
- 音频/视频文件路径会传递给生成器处理

## 九、3.6 模板结构（32页）

| 页码 | 类型 | 占位符 | 图片数 |
|------|------|--------|--------|
| 1 | 封面 | `{{h0_0}}` | 1 |
| 2 | 目录 | `{{h1_0}}~{{h1_9}}` | 2 |
| 3 | 章节 | `{{h1_0}}` | 0 |
| 4-11 | 正文+视频/音频 | `{{h2_0}}`+`{{h3_0}}`+`{{audio}}/{{video}}` | 0-5 |
| 12-18 | 正文+音频 | `{{h2_0}}`+`{{h3_*}}`+`{{audio}}` | 0-4 |
| 19-24 | 正文 | `{{h2_0}}`+`{{h3_0}}` | 0-5 |
| 25-28 | 双栏正文 | `{{h2_0}}`+`{{h3_0}}`+`{{h3_1}}` | 0-4 |
| 29-31 | 多栏正文 | `{{h2_0}}`+`{{h3_0}}~{{h3_3}}` | 1-4 |
| 32 | 结束 | 谢谢/再见 | 2 |

## 十、完整流程示例

**输入MD：**
```markdown
# 颜色世界

## 白色

### 学一学
A：这是什么颜色？||B：这是白色。

### 词汇
白
白衣服

## 红色

### 学一学
A：这是红色吗？||B：是的，这是红色。
```

**输出PPT结构：**
1. 封面页：颜色世界
2. 目录页：白色、红色
3. 章节页：白色
4. 正文页：学一学（2个文本框）
5. 正文页：词汇（2个文本框）
6. 章节页：红色
7. 正文页：学一学（2个文本框）
8. 结束页：谢谢
