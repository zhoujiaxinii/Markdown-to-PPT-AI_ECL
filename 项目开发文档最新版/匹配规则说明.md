# 匹配规则说明

> 更新日期：2026-03-11
> 基于 3.10 模板.pptx (37页)
> 详细说明MD内容如何匹配到PPT模板的规则

---

## 一、PPT结构

生成的PPT结构如下：

```
1. 封面页 - 课程标题 ({{h0_0}}) + 1图片
2. 章节页 - 每个二级标题一页 ({{h1_0}})
3. 正文页 - 每个三级标题一页 ({{h2_0}} + {{h3_*}})
4. ...
5. 结束页 - 谢谢（自动添加，无需MD编写） + 2图片
```

---

## 二、智能模板匹配

### 2.1 匹配原则

**核心要求**：根据MD内容精确匹配模板，清除多余占位符

| MD内容 | 匹配模板 | 说明 |
|--------|---------|------|
| 课程标题 | 封面模板 | `{{h0_0}}` |
| 二级标题 | 章节模板 | `{{h1_0}}` |
| 三级标题 | 正文模板 | 根据文本框数+图片数+媒体类型匹配 |
| 含音频 | 音频模板 | 自动识别 `{{audio}}` |
| 含视频 | 视频模板 | 自动识别 `{{video}}` |
| 含链接 | 链接模板 | 自动识别 `{{link}}` |
| N个文本框 | 文本框数≥N的最小模板 | 清除未使用的占位符 |
| M个图片 | 图片数=M的模板 | 精确匹配图片数 |

### 2.2 媒体类型优先级

当一页同时包含多种媒体时，按以下优先级匹配：
```
video > audio > link > None
```

### 2.3 页面生成顺序

```
1. 解析MD文档
   ├─ 提取课程标题 (h0)
   ├─ 提取章节列表 (h1)
   └─ 提取小节内容 (h2 + content + audio/video/link)

2. 生成PPT页面
   ├─ 封面页 (h0)
   ├─ 遍历章节
   │   ├─ 章节页 (h1标题)
   │   └─ 正文页 (h2内容)
   └─ 结束页 (自动添加)

3. 清理页面
   └─ 清除所有包含{{...}}的未替换占位符
```

## 三、占位符规则

### 3.1 文本占位符

| 占位符 | 用途 |
|--------|------|
| `{{h0_0}}` | 封面标题 |
| `{{h1_0}}` | 章节标题 |
| `{{h2_0}}` | 小节标题 |
| `{{h3_0}}`~`{{h3_3}}` | 文本内容（最多4个） |

### 3.2 媒体占位符

| 占位符 | 用途 |
|--------|------|
| `{{audio}}` | 音频图标位置，点击可播放 |
| `{{video}}` | 视频播放器位置 |
| `{{link}}` | 链接位置，点击后用浏览器打开 |

### 3.3 模板匹配示例

| 小节内容 | 匹配模板 |
|---------|---------|
| 1个文本框 | 单文本框模板 |
| 2个文本框 | 双文本框模板 |
| 3个文本框 | 三文本框模板 |
| 4个文本框 | 四文本框模板 |
| 含音频 | 匹配带 `{{audio}}` 的模板 |
| 含视频 | 匹配带 `{{video}}` 的模板 |
| 含链接 | 匹配带 `{{link}}` 的模板 |

## 四、章节页处理

### 4.1 章节识别

MD中的二级标题 (`##`) 会被识别为章节：

```markdown
## 白色    ← 章节页：白色
### 学一学  ← 正文页
### 词汇    ← 正文页

## 红色    ← 章节页：红色
### 学一学  ← 正文页
```

### 4.2 章节页生成

- 每个二级标题生成一个章节页
- 使用模板中的 `{{h1_0}}` 占位符
- 章节名称自动添加拼音

---

## 五、结束页处理

### 5.1 自动添加

- PPT末尾自动添加结束页
- 无需在MD中编写
- 模板中应包含"谢谢"或"xiè xie"字样

### 5.2 模板识别

结束页通过以下方式识别：
- 页面中包含"谢谢"文字
- 页面中包含"xiè xie"拼音

## 六、拼音生成规则

### 6.1 垂直对齐原则
- **核心要求**：每个拼音精确对齐到对应汉字的**正上方**
- 拼音和汉字**一一对应**

### 6.2 表格实现方案
使用 **2行 x N列** 的无边框、无底色表格：

```
原文：这是白色
实现：
 zhè  shì  bái  sè
 这    是    白    色
```

### 6.3 英文处理
- 英文文本保留单词间的空格
- 纯英文行使用单行单列表格（合并单元格）

### 6.4 字符间距规则
- **拼音间距**：每个拼音后 + 1个字母宽度
- **汉字间距**：跟随对应拼音
- **列宽计算**：动态计算，不均分

### 6.5 自动缩放
- 文本过长时自动缩小字号
- 最小字号：10号

## 七、媒体处理

### 7.1 音频识别

MD中以 `.mp3`、`.wav`、`.m4a` 结尾的URL被识别为音频：
```markdown
https://example.com/audio.mp3
```

**处理规则**：
- 音频可以单独占一页，也可以与文本框同页
- 匹配含 `{{audio}}` 占位符的模板页
- 嵌入音频，显示图标，点击可播放

### 7.2 视频识别

MD中以 `.mp4`、`.avi`、`.webm` 结尾的URL被识别为视频：
```markdown
https://example.com/video.mp4
```

**处理规则**：
- 视频可以单独占一页，也可以与文本框同页
- 匹配含 `{{video}}` 占位符的模板页
- 嵌入视频，提取第一帧作为封面

### 7.3 链接识别

MD中的任何URL被识别为链接（当没有音视频时）：
```markdown
https://example.com/link
```

**处理规则**：
- 匹配含 `{{link}}` 占位符的模板页
- 点击后用默认浏览器打开

## 十、排版格式对应关系

### 10.1 模板占位符 → PPT呈现

| 模板占位符 | 内容示例 | PPT最终呈现 |
|-----------|----------|------------|
| `{{h0_0}}` | 颜色世界 | 48pt黑体，居中显示 |
| `{{h1_0}}` | 白色 | 66pt黑体，完全居中 |
| `{{h2_0}}` | 学一学 | 28pt黑体，保持模板位置 |
| `{{h3_0}}` | 你好吗？ | 拼音表格，汉字上方显示拼音 |
| `{{h3_0}}` | I am fine. | 纯英文表格，合并单元格 |
| `{{audio}}` | audio.mp3 | 嵌入音频，图标显示，点击可播放 |
| `{{video}}` | video.mp4 | 嵌入视频+封面 |
| `{{link}}` | https://... | 超链接，点击用浏览器打开 |

### 10.2 文本框数量 → 字体大小

| MD文本框数 | 条件 | 字体大小 |
|------------|------|----------|
| 1个 | 中文≤10 | 40pt |
| 1个 | 中文>10 | 28pt |
| 2个 | 中文>10 | 28pt |
| 2个 | 其他 | 36pt |
| 3个 | - | 30pt |
| 4个 | - | 24pt |

## 十一、完整流程示例

**输入MD：**
```markdown
# 颜色世界

## 白色

### 学一学
A：这是什么颜色？||B：这是白色。

### 听录音
请听录音判断对错。

https://example.com/audio.mp3

### 点一点
点击玩游戏。

https://example.com/game
```

**输出PPT结构：**
1. 封面页：颜色世界
2. 章节页：白色
3. 正文页：学一学（2个文本框）
4. 正文页：听录音（1个文本框+音频）
5. 正文页：点一点（1个文本框+链接）
6. 结束页：谢谢
