# 开发需求

> 更新日期：2026-03-05

## 项目信息
- 项目名称：Markdown to PPT AI_ECL
- 项目目标：将 Markdown 文档自动转换为带拼音标注的 PPT 课件
- 使用场景：Echineselearning 对外汉语教学平台

---

## 整体逻辑

```
输入：1个 Markdown 文件 + 1个 PPT 模板
输出：1个填充好内容的 PPT 文件

流程：
┌─────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ MD 文件  │ →  │ MD解析器  │ →  │ PPT生成器 │ →  │ 输出PPT  │
└─────────┘    └──────────┘    └──────────┘    └──────────┘
                    ↑                ↑
                解析3级标题       读取PPT模板
                提取文本/图片     匹配模板页
                识别音视频       填充拼音表格
```

## R1：Markdown 解析

### 需求描述
将 Markdown 文件解析为结构化数据，供 PPT 生成器使用。

### 规则

| MD语法 | 含义 | PPT对应 |
|--------|------|---------|
| `# 标题` | 课程标题 | 封面页标题 |
| `## 标题` | 章节名 | 章节页 |
| `### 标题` | 小节名 | 正文页标题 |
| `文本内容` | 正文 | 正文页文本框 |
| `文本A\|\|文本B` | 多文本框 | 同一页内多个文本框（最多4个） |
| `![图片](url)` | 图片 | 嵌入图片（最多5张/页） |
| `![音频](url)` | 音频文件 | 嵌入音频 |
| `![视频](url)` | 视频文件 | 嵌入视频 |

### 实现效果
输入 Markdown：
```markdown
# 颜色世界
## 白色
### 学一学
A：这是什么颜色？||B：这是白色。
```
输出数据结构：
```python
{
    'h0': ['颜色世界'],
    'h1': ['白色'],
    'h2': [{
        'title': '学一学',
        'section': '白色',
        'content': ['A：这是什么颜色？', 'B：这是白色。'],
        'images': [], 'video': None, 'audio': None
    }]
}
```

## R2：PPT 页面结构

### 需求描述
生成的 PPT 必须按固定结构排列页面。

### 页面顺序规则

```
封面页（1页）
  └─ 显示课程标题（# 标题）

章节 A 的内容
  ├─ 章节页（1页）── 显示章节名
  ├─ 正文页 1 ── 第一个 ### 小节
  ├─ 正文页 2 ── 第二个 ### 小节
  └─ ...

章节 B 的内容
  ├─ 章节页（1页）
  ├─ 正文页 ...
  └─ ...

结束页（1页）
  └─ 自动添加，显示"谢谢"（无需MD编写）
```

## R3：拼音标注与对齐

### 需求描述
所有汉字自动生成拼音，拼音显示在汉字**正上方**，一一精确对齐。

### 实现方案
使用 **2行 × N列透明表格**：
- 第1行：拼音（Arial字体）
- 第2行：汉字（SimSun字体）
- 表格无边框、无背景色
- 列宽根据拼音长度动态计算
- 字号超出宽度时自动缩小（最小10pt）
- 单元格禁止换行

### 实现效果
```
 zhè  shì  bái  sè
 这    是    白    色
```
在 PPT 中看起来就像拼音自然标注在汉字上方，看不到表格的存在。

## R4：智能模板匹配

### 需求描述
根据每页 MD 内容的**文本框数量**，自动从模板中选择最合适的页面。

### 匹配规则

| 页面类型 | 识别方式 | 占位符 |
|---------|---------|--------|
| 封面页 | 含 `{{h0_0}}` | h0_0 |
| 章节页 | 含 `{{h1_0}}`，不含 `{{h2_0}}` | h1_0 |
| 正文页 | 含 `{{h2_0}}` | h2_0 + h3_0~h3_3 |
| 结束页 | 含"谢谢"或"xiè xie" | 无 |

**正文页匹配逻辑**：
- 按 `{{h3_*}}` 占位符数量分类（1框、2框、3框、4框）
- MD 内容有 N 个文本框 → 选 ≥N 的最小模板
- 多余的占位符自动清除

### 实现效果
MD 中一个小节有2个文本框 → 自动选择含2个 `{{h3_*}}` 的模板页，填充内容后清除多余占位符，页面干净无残留。

## R5：模板背景保留

### 需求描述
生成的 PPT 必须保留模板的**背景、图片、装饰元素**。

### 实现方案
- 在模板 PPT 上直接操作（不创建新PPT）
- 需要复用的模板页通过 **XML 深拷贝** 克隆
- 克隆保留：背景（来自slide layout）、图片、自动形状等
- 先克隆后填充，避免克隆到已填充的内容
- 用 rId 追踪每个幻灯片，确保删除和排序不出错

### 实现效果
输出的 PPT 每一页都保留模板原有的背景色、装饰图片、排版样式，只有占位符位置被替换为带拼音的文本内容。

## 内容限制

| 元素 | 限制 | 说明 |
|------|------|------|
| 文本框/页 | 最多4个 | 用 `\|\|` 分隔 |
| 图片/页 | 最多5张 | `![图片](url)` |
| 标题层级 | 3级 | `#` → `##` → `###` |
| 拼音最小字号 | 10pt | 超宽时自动缩小 |
