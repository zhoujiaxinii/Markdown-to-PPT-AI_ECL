## 一、完整需求（10点）

| # | 需求 | 说明 |
|---|------|------|
| 1 | 标题3级 | 课程标题 `#` → 章节名 `##` → 小节名 `###`，每页都有小节名 |
| 2 | 字数限制提示 | 每个小节后显示字数范围，建议文本框数量 |
| 3 | 图片0-5张/页 | 支持0-5张图片 |
| 4 | 文本框1-4个/页 | 用 `\|\|` 分隔多文本框 |
| 5 | 视频(mp4)单独一页 | 格式：`![视频](url)` |
| 6 | 音频(mp3)+文本同页 | 格式：`![音频](url)`，匹配带喇叭图标模板 |
| 7 | 汉字带拼音 | 拼音在汉字**上方**，字号=汉字字号 |
| 8 | 同级标题大小统一 | `##` 统一字号，`###` 统一字号 |
| 9 | 字体大小灵活 | 最大60号，根据页面空白调整 |
| 10 | 输出新PPT模板 | 保持底板，增加占位符和喇叭图标 |

## 二、MD撰写规范摘要

### 2.1 基本结构
```markdown
# 课程标题

## 章节名

### 小节名

文本内容||文本内容2

![图片](url)
![图片1](url)

![音频](url)

![视频](url)
```

### 2.2 关键规则
- 每个 `### 小节名` 生成一页PPT
- 用 `||` 分隔多文本框（最多4个）
- 用 `![图片](url)` 添加图片（最多5张）
- 用 `![音频](url)` 添加音频
- 用 `![视频](url)` 添加视频

## 五、技术实现要点

### 5.1 多文本框支持
- 使用 `||` 作为分隔符
- 解析后生成 `h3_0`, `h3_1`, `h3_2`, `h3_3` 四个占位符

### 5.2 图片支持
- 支持0-5张图片
- 对应 `img_0` ~ `img_4` 占位符

### 5.3 音视频支持
- 音频：识别 `![音频](url)` 或 `.mp3` 后缀
- 视频：识别 `![视频](url)` 或 `.mp4` 后缀
- 音频使用带喇叭图标的模板
- 视频单独一页显示

### 5.4 拼音支持
- 使用 pypinyin 库
- 拼音显示在汉字上方
- 字号为汉字字号的一半
