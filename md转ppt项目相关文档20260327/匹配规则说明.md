# 匹配规则说明

> 更新日期：2026-03-27
> 基于代码版本：V16
> 详细说明 MD 内容如何匹配到 PPT 模板

---

## 一、PPT 页面结构

```
1. 封面页 — 课程标题 ({{h0_0}})
2. 章节1 — 章节名 ({{h1_0}})
3. 正文页 — 小节1 ({{h2_0}} + {{h3_*}})
4. 正文页 — 小节2 ({{h2_0}} + {{h3_*}})
5. 章节2 — 章节名 ({{h1_0}})
6. 正文页 ...
7. 结束页 — 谢谢（自动添加）
```

---

## 二、模板分析（_analyze_templates）

代码启动时扫描所有模板页，建立索引：

### 2.1 页面类型识别

| 类型 | 识别条件 | 存储 |
|------|---------|------|
| 封面 | 包含 `{{h0_0}}` | `self.templates['cover'] = 页索引` |
| 章节 | 包含 `{{h1_0}}` 且不含 `{{h2_0}}` | `self.templates['section'] = 页索引` |
| 结束 | 包含"谢"或"xiè" | `self.templates['end'] = 页索引` |
| 正文 | 包含 `{{h2_0}}` | 加入三维索引 |

### 2.2 正文三维索引

```python
self.content_index = {
    (h3_count, img_count, media_type): [页索引列表]
}
```

**h3_count**: `{{h3_*}}` 占位符数量（0-4）
**img_count**: 图片 Shape 数量
**media_type**: 检测媒体占位符
- `'video'` — 含 `{{video}}`
- `'audio'` — 含 `{{audio}}`
- `'link'` — 含 `{{link}}`
- `None` — 无媒体占位符

---

## 三、模板匹配（_match_content_template）

对每页 MD 内容，按以下优先级匹配：

```
步骤1: 精确匹配 (text_count, image_count, media_type)
  → 找到？返回模板页列表
  
步骤2: 忽略媒体类型匹配 (text_count, image_count, None)
  → 找到？返回模板页列表
  
步骤3: 无匹配
  → 返回空列表
  → 调用方使用第一个正文模板 + 左上角红色提示
```

**媒体类型优先级**（用于MD内容判断）：
```
video > audio > link > None
```
一页内容只取最高优先级的媒体类型进行匹配。

---

## 四、轮换机制

同一模板页被多次匹配时，通过 `_clone_slide()` XML深拷贝复用。

同一组合（如 1文本框 0图片）有多个模板页时，**轮换使用**：
```python
ptr = {}  # (模板列表) → 当前索引
# 第1次使用 → 模板A
# 第2次使用 → 模板B
# 第3次使用 → 模板A（循环）
```

## 五、内容填充流程

### 5.1 封面页
```python
_fill(slide, 'h0_0', title, fs=48)
_center_horizontal(slide, 'h0_0')  # 水平居中，垂直位置不变
```

### 5.2 章节页
```python
_fill(slide, 'h1_0', section_name, fs=66)
_center_text(slide, 'h1_0')  # 完全居中（水平+垂直）
```

### 5.3 正文页
```python
_fill(slide, 'h2_0', title, fs=28, bold=True)  # 小节标题

# 媒体处理（优先级：link → audio → video）
if link: _embed_link(slide, link)
if audio: _embed_media_to_placeholder(slide, 'audio', audio)
if video: _embed_media_to_placeholder(slide, 'video', video)

# 文本填充（按字号规则）
for j, text in enumerate(content_list):
    _fill(slide, f'h3_{j}', text, fs)

# 图片替换
_replace_images_on_slide(slide, images)

# 清除未使用的占位符
_clear_unused(slide, used_placeholders)
```

## 六、拼音生成规则

### 6.1 中文内容 → 2行×N列表格

```
 zhè  shì  bái  sè
 这    是    白    色
```

- 第1行：拼音（SimHei，底部对齐）
- 第2行：汉字（SimHei，顶部对齐）
- 列宽动态计算，表格无边框无底色

### 6.2 纯英文内容 → 1行×1列表格

```
 I am fine.
```

- 合并单元格
- 保留空格
- 字号固定 22pt

### 6.3 混合内容

- 连续英文段合并为一个单元格
- 中文字符独立一个单元格
- 拼音行英文段留空

## 七、字号规则

| 文本框数 | 条件 | 字号 |
|----------|------|------|
| 1个 | 无图片，中文≤10字 | 40pt |
| 1个 | 无图片，中文>10字 | 28pt |
| 2个 | 无图片，中文>10字 | 28pt |
| 2个 | 其他 | 36pt |
| 3个 | - | 30pt |
| 4个 | - | 24pt |

自动缩放：文本超出文本框时缩小（最小8pt）。

## 八、媒体处理

### 8.1 音频
- 检测 `.mp3`/`.wav`/`.m4a` 结尾的 URL
- 匹配含 `{{audio}}` 占位符的模板
- 嵌入到占位符位置（`add_movie`）
- 在线URL下载到 `.tmp_media/` 缓存

### 8.2 视频
- 检测 `.mp4`/`.avi`/`.webm` 结尾的 URL
- 匹配含 `{{video}}` 占位符的模板
- ffmpeg 提取第一帧作封面（0.5秒处）
- 嵌入到占位符位置（`add_movie` + `poster_frame_image`）

### 8.3 链接
- 非音视频 URL 识别为链接
- 匹配含 `{{link}}` 占位符的模板
- 使用 `click_action.hyperlink.address` 添加超链接
- 显示蓝色文本："🔗 点击打开游戏链接进行游玩"

## 九、完整流程示例

**输入MD：**
```markdown
# 颜色世界

## 白色

### 学一学
A：这是什么颜色？||B：这是白色。

### 听录音
请听录音判断对错。
https://example.com/audio.mp3

### 点一点
点击玩游戏。
https://example.com/game
```

**匹配过程：**

| 页面 | 内容 | 匹配 |
|------|------|------|
| 封面 | "颜色世界" | 封面模板 `{{h0_0}}` |
| 章节 | "白色" | 章节模板 `{{h1_0}}` |
| 正文 | 学一学，2框0图 | 匹配 `(2, 0, None)` 模板 |
| 正文 | 听录音，1框0图+音频 | 匹配 `(1, 0, 'audio')` 模板 |
| 正文 | 点一点，1框0图+链接 | 匹配 `(1, 0, 'link')` 模板 |
| 结束 | 谢谢 | 结束模板 |

**输出PPT：** 6页（封面 + 章节 + 3页正文 + 结束）
