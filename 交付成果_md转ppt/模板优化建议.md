# 模板优化建议

## 1. 占位符命名规范化（建议）

当前模板已使用 `h0/h1/h2/h3...`，建议继续统一：

- 封面：`h0_0`
- 目录：`h1_0...h1_n`
- 章节：`h1_0`
- 小节标题：`h2_0`
- 正文文本：`h3_0...h3_n`
- 图片URL占位：`h4_0...h4_n`
- 音频URL占位：`h5_0...h5_n`
- 视频URL占位：`h6_0...h6_n`

## 2. 建议补充的模板页类型

建议在模板中额外增加以下“高频版式”，提高泛化：

1. 纯文本 1列（1标题+6~10条正文）
2. 左文右图（1图）
3. 左文右双图（2图）
4. 四宫格图文（4图+4短文）
5. 媒体说明页（标题+媒体链接+提示文）
6. 代码讲解页（标题+大文本框等宽字体）
7. 对比页（左右两栏）

## 3. 增加“机器可识别标识”

为每个内容模板页新增隐藏文本框，写唯一标识，如：

- `layout:text_3`
- `layout:text_image_1`
- `layout:gallery_4`
- `layout:media`

这样匹配将不依赖启发式评分，可直接精准命中。

## 4. 目录与章节容量建议

- 目录页建议预留至少 8~12 个 `h1_i`
- 正文页建议至少覆盖 `h3_0`~`h3_5`
- 图文页建议预留明确的 1/2/3/4 图版本

## 5. 样式一致性建议

- 同类型占位符字体、字号、行距保持一致
- 图片容器比例统一（如 1:1 或 4:3）
- 对链接文字启用自动换行
- 保留足够边距，避免长URL溢出

## 6. 音视频现实限制

python-pptx 对“可播放媒体嵌入”的支持有限，当前更稳妥方式是：

- 在PPT中放置媒体URL文本
- 或预留图标占位，点击跳转

如果必须离线播放，需要额外的COM自动化或Office宏链路。
