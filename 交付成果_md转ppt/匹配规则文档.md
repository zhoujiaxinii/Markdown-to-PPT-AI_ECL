# 匹配规则文档（基于示例《说白色》）

## 0. 拼音-汉字对齐规则（表格方案）

### 实现原理
使用 **2行 x N列无边框、无底色表格** 实现拼音与汉字的精确对齐：

```
 zhè  shì  bái  sè
 这    是    白    色
```

### 字符间距规则
- **拼音间距**：每个拼音后 + 1个字母宽度的间距
- **汉字间距**：跟随对应拼音，自动对齐
- **列宽计算**：根据拼音长度动态计算（不再均分）

### 技术细节
- **表格结构**：2行 × N列（N = 字符数）
- **边框**：完全隐藏（无边框）
- **底色**：透明（无填充）
- **字体**：拼音用 Arial，汉字用 SimSun
- **字号**：拼音和汉字相同
- **行高**：拼音行45%，汉字行55%
- **对齐**：每个单元格内水平居中、垂直居中

### 优势
- 每个拼音-汉字对在独立单元格中，保证100%精确对齐
- 动态列宽，字符间距紧凑自然
- 无边框、无底色，视觉效果干净

## 1. 页面角色识别规则

工具先扫描模板中每一页的占位符与固定文字，给模板页打角色标签：

- 封面页：包含 `{{h0_0}}`
- 目录页：包含“目录”或“mù lù”固定字样
- 章节页：包含 `{{h1_0}}` 且不包含 `{{h2_0}}`
- 内容页：包含 `{{h2_0}}`
- 结束页：包含“xiè xie”或“谢”固定字样

## 2. Markdown结构到PPT结构映射

- `# 标题` → 封面标题（填入 `h0_0`）
- `## 章节` → 章节页标题（填入 `h1_0`），并作为目录项
- `### 小节` → 一页内容页标题（填入 `h2_0`）
- 该 `###` 下的文本、列表、代码、链接、音视频、图片 → 内容页正文

## 3. 特殊编排规则（还原示例习惯）

- 若第一个 `##` 章节是“今天学什么”/“导入”或块数较多，判定为导入段：
  - 导入块排在目录前（和示例一致：封面后先出现导入内容，再目录）
- 其他章节按“章节页 + 若干内容页”顺序生成
- 末尾自动添加结束页（模板中存在时）

## 4. 内容页布局自动选择评分规则

对每个 `###` 内容块，统计：

- 文本需求：文本行 + 列表项 + 代码块 + 链接/音频/视频行
- 图片需求：图片数量
- 多媒体需求：音频+视频+链接数量

对每个候选内容模板页计算得分：

- 文本槽位足够：高分，且越接近需求越好
- 文本槽位不足：按缺口扣分
- 需要多图（>=3）且模板自带多图片区：加分
- 无图片却选到重图片区模板：扣分
- 有多媒体且模板文本槽位较多：加分
- 有代码块且文本槽位>=2：加分

取分数最高的模板页。

## 5. 占位符填充规则

- `h2_0`：填 `###` 标题
- `h3_i`：按顺序填文本行（正文、列表、代码、链接等）
- `h4_i`：优先填图片URL；若该页有图片shape，则尝试下载图片并替换到对应图片区
- `h5_i`：音频URL
- `h6_i`：视频URL
- 未用占位符清空
